# ClearCache1C
Графическая утилита для очистки кэша 1С (с возможностью выбора базы) для ОС Linux
# Приложение для очистки кэша 1С Предприятие

Это графическое приложение на базе PyQt6, которое предоставляет простой интерфейс для очистки локального кэша баз данных 1С Предприятие.

## Особенности

- Автоматическое обнаружение баз данных 1С из конфигурационного файла
- Выпадающий список для выбора базы данных
- Кнопка "Очистить" для запуска очистки кэша
- Кнопка "Выход" для закрытия приложения
- Автоматическое завершение процессов 1С перед очисткой
- Автоматическое закрытие сообщения об успехе через 5 секунд

## Требования

- Python 3.6 или выше
- Библиотека PyQt6
- Операционная система Linux (для команды завершения процессов pkill)
- Файл конфигурации баз данных 1С (ibases.v8i)

## Установка

Перед запуском приложения необходимо установить PyQt6. В системах с соответствующими правами доступа:

```bash
pip install PyQt6
```

В системах Debian/Ubuntu может потребоваться использовать:

```bash
sudo apt-get install python3-pip
pip3 install PyQt6
```

В системах, где у вас нет прав администратора, вы можете попробовать установить в пользовательское пространство:

```bash
pip3 install --user PyQt6
```

Или использовать conda, если она доступна:

```bash
conda install pyqt
```

## Использование

Запустите приложение с помощью:

```bash
python3 main.py
```

## Создание автономного исполняемого файла

Чтобы создать автономный исполняемый файл, который может работать без установленного Python, используйте скрипт сборки:

```bash
./build.sh
```

Это создаст автономный исполняемый файл в папке `dist`.

Примечание: Если вы столкнетесь с проблемами при установке пакетов из-за системных ограничений, возможно, вам потребуется использовать виртуальное окружение:

```bash
python3 -m venv cache_cleaner_env
source cache_cleaner_env/bin/activate  # В Windows: cache_cleaner_env\\Scripts\\activate
pip install PyQt6 pyinstaller
./build.sh
deactivate
```

В качестве альтернативы, вы можете вручную создать исполняемый файл с помощью PyInstaller:

```bash
pip install pyinstaller PyQt6
pyinstaller ClearCache1C.spec
```

## Структура приложения

Окно приложения содержит:
1. Логотип с ссылкой на техническую поддержку
2. Метку с просьбой выбрать базу для очистки
3. Выпадающее меню со списком обнаруженных баз данных 1С
4. Информационную метку с предупреждением (красный цвет)
5. Две кнопки: "Очистить" и "Выход"

## Функциональность

Приложение **ClearCache1C** предоставляет удобный графический интерфейс для очистки локального кэша баз данных 1С Предприятие. Ниже представлено подробное описание всех функций и принципов работы приложения.

### Автоматическое обнаружение баз данных 1С

При запуске приложение автоматически выполняет поиск конфигурационных файлов 1С в домашнем каталоге пользователя. Оно анализирует файл `ibases.v8i`, расположенный по пути `~/.1C/1cestart/ibases.v8i`. Из этого файла приложение извлекает список зарегистрированных информационных баз и их идентификаторы.

Для каждой обнаруженной базы данных приложение проверяет наличие соответствующего каталога кэша по пути `~/.1cv8/1C/1cv8/<ID_базы>`. Только те базы, для которых существует каталог кэша, отображаются в выпадающем списке. Это гарантирует, что пользователь может очистить только реально существующий кэш.

Список баз отображается в выпадающем списке в алфавитном порядке для удобства навигации. Если на компьютере не обнаружено ни одной настройки 1С, в выпадающем списке отображается сообщение «Не обнаружены настройки 1С», а кнопка «Очистить» деактивирована.

### Интерфейс приложения

Окно приложения содержит следующие элементы:

- **Логотип** — изображение отображается в левой части окна для визуальной идентификации приложения
- **Ссылка на техническую поддержку** — расположена под логотипом, открывает страницу технической поддержки
- **Заголовок окна** — «Очистка локального кэша баз 1С Предприятие»
- **Метка выбора** — «Выберите базу для очистки»
- **Выпадающий список** — содержит список обнаруженных баз данных 1С
- **Информационная метка** — отображает предупреждение «Сохраните свои данные, все базы 1С будут закрыты» (красный цвет)
- **Кнопка «Очистить»** — запускает процесс очистки кэша (активна только при наличии обнаруженных баз)
- **Кнопка «Выход»** — закрывает приложение

### Процесс очистки кэша

При нажатии кнопки «Очистить» приложение выполняет следующую последовательность действий:

1. **Подтверждение пользователя** — отображается диалоговое окно с вопросом «Вы хотите продолжить очистку: [название базы]?». Пользователь может подтвердить или отменить операцию. Кнопки «Нет» и «Да» расположены в указанном порядке.

2. **Завершение процессов 1С** — перед очисткой кэша приложение автоматически завершает все запущенные процессы 1С Предприятие (1cv8 и 1cv8c) с помощью команды `pkill`. Это необходимо для освобождения файлов кэша и предотвращения конфликтов.

3. **Определение пути к кэшу** — на основе идентификатора выбранной базы данных формируется путь к каталогу кэша в формате `~/.1cv8/1C/1cv8/<ID_базы>`.

4. **Удаление каталога кэша** — весь каталог кэша выбранной базы данных удаляется с помощью рекурсивного удаления (`shutil.rmtree`).

5. **Уведомление об успешном завершении** — после успешного завершения всех операций отображается информационное сообщение «Очистка кэша завершена успешно!». Это окно автоматически закрывается через 5 секунд.

### Обработка ошибок

Приложение обрабатывает следующие типы ошибок:

- **Ошибка чтения настроек 1С** — если файл `ibases.v8i` не может быть прочитан, ошибка выводится в консоль
- **Ошибка удаления кэша** — если каталог кэша не может быть удален, отображается диалоговое окно с описанием ошибки
- **Ошибка завершения процессов** — если процессы 1С не могут быть завершены, ошибка выводится в консоль

- Операционная система Linux (для команды завершения процессов pkill)

### Константы приложения

Приложение использует следующие константы для настройки:

- `TECH_SUPPORT_URL` — URL страницы технической поддержки
- `WINDOW_WIDTH` — ширина окна (600 пикселей)
- `WINDOW_HEIGHT` — высота окна (250 пикселей)
- `LOGO_WIDTH` — ширина логотипа (300 пикселей)
- `LOGO_HEIGHT` — высота логотипа (168 пикселей)
- `AUTO_CLOSE_TIMEOUT_MS` — время автоматического закрытия сообщения об успехе (5000 мс)
- `IBASES_PATH` — путь к файлу конфигурации баз данных (`.1C/1cestart/ibases.v8i`)
- `CACHE_DIR_PATH` — путь к каталогу кэша (`.1cv8/1C/1cv8`)
